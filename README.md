# AI Shorts Story Factory

A GitHub-ready guide for building a faceless YouTube Shorts content system using original fictional confession-style stories, AI voice, Minecraft/gameplay backgrounds, big captions, and repeatable prompts.

> Goal: build a repeatable AI-assisted Shorts workflow that can generate, polish, produce, schedule, and improve short-form videos with minimal daily time.

---

## Project Overview

This project is a faceless short-form content engine focused on:

* Original fictional confession stories
* Creepy drama / school drama / betrayal / mystery formats
* Minecraft parkour or gameplay backgrounds
* AI-generated scripts
* AI voice narration
* Auto captions
* Fast editing templates
* Daily upload consistency

This is designed for beginners who want a simple content system without clients, calls, sales, or showing their face.

# Full Workflow

## Step 1: Pick a Theme

Do not tell AI to generate “any story.” Give it a theme.

Good themes:

```txt
creepy school stories
best friend betrayal
roommate acting weird
dog knew something was wrong
3AM phone call
hidden camera discovery
neighbor watching
teacher exposed secret
friend disappeared overnight
family secret
school group chat exposed
```

Example command:

```txt
Create 10 original fictional confession-style YouTube Shorts stories.

Theme: school group chat exposed
Audience: teenagers
Style: dramatic, realistic, high-retention, comment-bait
Length: 30–45 seconds
Use short punchy lines.
End each with a twist and a question.
```

---

## Step 2: Generate Stories

Use the **One-Click Factory Prompt** to generate a batch of scripts.

Generate 10–20 at once.

Do not use every output.

Pick only the best 1–3.

---

## Step 3: Upgrade the Best Stories

Run the best story through:

```txt
Make It 10x Better Prompt
```

This improves:

* Hook
* Pacing
* Red flags
* Twist
* Final payoff
* Comment bait

---

## Step 4: Run Retention Check

Use the **Retention Checker Prompt**.

Only produce videos from scripts that score at least:

```txt
7/10 or higher
```

If the score is low, regenerate or improve.

---

## Step 5: Generate Voice

Use an AI voice tool.

Recommended voice style:

```txt
calm but suspenseful
fast but clear
emotional but not cringe
natural storyteller voice
```

Export as:

```txt
.mp3
```

---

## Step 6: Add Gameplay Background

Best gameplay backgrounds:

* Minecraft parkour
* Subway Surfers
* GTA driving
* Rocket League
* Satisfying mobile gameplay

Prefer copyright-safe or self-recorded gameplay.

Avoid relying on stolen clips long-term.

---

## Step 7: Edit in CapCut

Timeline structure:

```txt
Gameplay Background
+ AI Voiceover
+ Auto Captions
+ Low-Volume Suspense Music
+ Zoom Effects
+ Fast Cuts
```

Recommended video settings:

```txt
Aspect ratio: 9:16
Length: 30–45 seconds
Captions: large, readable, centered
Music: low volume
Voice: loud and clear
Cuts: every 1–2 seconds
```

---

## Step 8: Caption Style

Captions should be:

* Large
* Fast
* Easy to read
* Broken into short chunks
* Emphasized on key words

Example:

```txt
His DOG knew something was WRONG.

But nobody believed him...

Until 3AM.
```

Best words to emphasize:

```txt
3AM
DOG
GONE
LOCKED
HIDDEN
CAMERA
SECRET
WHISPERING
BEST FRIEND
TEACHER
GROUP CHAT
```

---

## Step 9: Upload

Upload to:

* YouTube Shorts
* TikTok
* Instagram Reels

Use the same username everywhere.

Recommended upload volume:

```txt
2–3 Shorts per day
```

Start simple.

Consistency beats perfection.

---

## Step 10: Analyze Results

Track:

* Views
* Average view duration
* Retention
* Likes
* Comments
* Shares
* Subscribers gained

The most important metric:

```txt
Retention
```

If people keep watching, platforms push the video harder.

For each story, output:

HOOK:
One powerful first sentence under 12 words.

SCRIPT:
A 30–45 second Shorts script with short caption-friendly lines.

TWIST:
Explain the twist in one sentence.

PAYOFF:
Explain why the ending is satisfying or disturbing.

CAPTION EMPHASIS:
List words that should be in ALL CAPS.

A comment-bait question viewers will answer.

Generate 10 stories.

# Story Formula

Use this structure every time:

```txt
Normal situation
→ small weird detail
→ second red flag
→ third red flag
→ character investigates
→ shocking discovery
→ final twist
→ unanswered question
```

Example skeleton:

```txt
He let his friend stay over.
The dog hated him.
Food disappeared.
The door kept opening.
At 3AM, he heard whispering.
His friend was talking to nobody.
Next morning, the friend vanished.
But his phone was still recording.
```
